# 适配器模式

---

## 核心内容

- 客户端想通过目标类的接口来访问目标类所提供的服务，但是它提供的接口不一定就是客户端所期望的
- 比如不同国家的电源插座（Adaptee），需要转接口（Adapter），充电器（Target/客户端）才能使用电源插座

## 正文

- 有两种实现方式：1.类适配器；2.对象适配器。一般使用对象适配器，因为规避了继承的局限性（[[为何组合优于继承]]）。
- 在Java等不允许多重继承的语言中，使用类适配器，一次只能适配一个Adaptee类。
- 适配器提供客户端需要的接口，当客户端调用适配器的方法时，适配器内部将调用Adaptee的方法，这样客户端无需担心接口不适合的问题，也不需要修改原有的客户端与Adaptee代码，符合开闭原则。

[[适配器模式-2025-11-23-08-53-30.svg]]
 ```plantuml-svg
 @startuml
 class Client{
 }
 
 interface Target{
	+request() 
 }
 
 class Adaptee{
	+SpecificRequest()
 }
 
 class Adapter{
	-Adaptee adaptee
	+request()
 }

 Client ..> Target: Dependency
 Adapter ..|> Target: implements
 Adapter --> Adaptee: Association
 @enduml
 ```
