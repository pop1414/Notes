# 可行性研究的任务

最新更新：`= dateformat(date(today), "yyyy-MM-dd")`

---

## 内容

- 目的：用最小的代价，在尽可能短的时间内确定问题是否能够解决
- 实质：就是一次压缩，简化了系统分析和设计的过程
- 路线：
	- 分析和澄清问题定义
	- 导出系统逻辑模型
	- 探索若干种可供选择的主要解法（系统实现方案）
	- 对每种解法进行可行性研究
	- 为每种可以的解法指定一个粗略的实现进度
- 可行性研究主要考虑以下几个方面
	- 技术可行性:使用先有的技术能否实现这个系统
	- 经济可行性：进行成本/效益分析，从经济角度判断系统开发是否“合算”
	- 操作可行性：系统的操作方式在这个用户组织内是否行得通
	- 法律可行性：确定系统开发可能导致的任何侵权、妨碍和责任
	- 开发方案的选择性选择：提出并评价实现系统的各种开发方案，并推荐较优方案
- 技术可行性：
	- 从开发者的技术实力、以往工作基础、问题的复杂性等出发，判断系统开发在时间、费用等限制条件下成功的可能性
	- 分析员需要根据系统的功能、性能需求，建立系统模型，然后对此模型进行一系列的实验、评审和修改。最后由项目管理人员作出是否进行系统开发的决定
	- 如果开发技术风险很大，或者模型演示表明当前采用的技术和方法不能实现系统预期的功能和性能，或者系统的实现不支持各子系统的集成，则项目管理人员可以作出停止系统开发的决定
- 经济可行性
	- 分析员需要进行成本/效益分析
	- 所谓成本：
		1. 购置并安装、硬件以及有关设备的费用
		2. 系统开发费用
		3. 系统安装、运行以及维护的费用
		4. 人员培训费用
	- 所谓效益：
		1. 系统为用户增加的收入或为用户节省的开支，这是有形的效益
		2. 给潜在用户心理上造成的影响，这是无形的效益。它可以转化为有形的效益
- 可行性研究最根本的任务：对以后的行动方针提出建议
	- 如果问题没有可以的解，应建议停止项目
	- 如果问题值得解，应推荐一个较好的解决方案，并为项目制定一个初步的计划
- 可行性研究的成本一般为预期总成本的5%~10%
